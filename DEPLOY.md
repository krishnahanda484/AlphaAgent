# Deploying AlphaAgent to Render

## Prerequisites

- A [Render](https://render.com) account
- A PostgreSQL database (Render provides managed Postgres, or use any external provider)
- An [OpenAI API key](https://platform.openai.com/api-keys) (for the GPT-4o research agent)

## Quick Deploy (Render Blueprint)

1. Push this project to a GitHub repository
2. Go to [Render Dashboard](https://dashboard.render.com)
3. Click **New** → **Blueprint**
4. Connect your GitHub repo
5. Render will detect `render.yaml` and auto-configure the service
6. Set the required environment variables (see below)
7. Click **Apply**

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `NODE_ENV` | ✅ | Set to `production` (auto-set by render.yaml) |
| `PORT` | ✅ | Set to `10000` (auto-set by render.yaml) |
| `DATABASE_URL` | ✅ | PostgreSQL connection string (e.g. `postgresql://user:pass@host:5432/dbname`) |
| `OPENAI_API_KEY` | ✅ | Your OpenAI API key for GPT-4o |
| `SESSION_SECRET` | ✅ | Auto-generated by Render |
| `BASE_PATH` | ❌ | Defaults to `/` — only change if serving under a subpath |

## What Happens During Build

The `render.yaml` build command runs:

```bash
corepack enable
pnpm install --no-frozen-lockfile
pnpm --filter @workspace/investment-agent run build    # Vite builds React frontend
pnpm --filter @workspace/api-server run build          # esbuild bundles Express server + copies frontend
pnpm --filter @workspace/db run push                   # Pushes the database schema
```

The API server build script (`artifacts/api-server/build.mjs`) automatically:
1. Bundles the Express server with esbuild into `artifacts/api-server/dist/index.mjs`
2. Copies the frontend build output from `artifacts/investment-agent/dist/` into `artifacts/api-server/dist/public/`

## What Happens at Runtime

```bash
node artifacts/api-server/dist/index.mjs
```

The Express server:
- Serves the API at `/api/*`
- Serves the React frontend static files from `dist/public/`
- Falls back to `index.html` for all other routes (SPA routing)
- Listens on `process.env.PORT` (Render provides this)

## Database Setup

The database schema is automatically pushed to your PostgreSQL database during the build phase. You do not need to run migrations manually, provided `DATABASE_URL` is set in the Render dashboard.

## Manual Deploy (Without Blueprint)

1. Create a **Web Service** on Render
2. Connect your GitHub repo
3. Set:
   - **Build Command**: `corepack enable && pnpm install --no-frozen-lockfile && pnpm --filter @workspace/investment-agent run build && pnpm --filter @workspace/api-server run build && pnpm --filter @workspace/db run push`
   - **Start Command**: `node artifacts/api-server/dist/index.mjs`
   - **Node Version**: 22+ (set `NODE_VERSION=22` in env vars if needed)
4. Add all required environment variables
5. Deploy

## Verify Deployment

After deployment:
- `https://your-app.onrender.com/` — should load the React app
- `https://your-app.onrender.com/api/healthz` — should return `{"status":"ok"}`
- `https://your-app.onrender.com/api/research/stats` — should return research statistics

## Troubleshooting

| Issue | Solution |
|---|---|
| Frontend not loading | Check that `artifacts/api-server/dist/public/index.html` exists in the build output |
| API returning 500 | Check that `DATABASE_URL` and `OPENAI_API_KEY` are set |
| Build fails with platform errors | Ensure `pnpm-lock.yaml` is not committed (it's regenerated on Render) |
| Port errors | Ensure `PORT` is set to `10000` |
